---

사용자가 제공한 내용으로 블로그 게시물을 작성하고 Git에 커밋/푸시하는 작업을 수행해주세요.

## 작업 절차

1. **날짜 확인**: 오늘 날짜를 YYYY-MM-DD 형식으로 확인
2. **파일명 생성**: `_posts/YYYY-MM-DD-{제목}.md` 형식으로 생성
3. **Front Matter 작성**: Jekyll 형식에 맞게 작성
   ```yaml
   ---
   title: 게시물 제목
   date: YYYY-MM-DD HH:MM:SS +0900
   categories: [카테고리명]
   tags: [태그1, 태그2]
   ---
   ```
4. **내용 작성**: 사용자가 제공한 마크다운 내용 작성
5. **Git 작업**:
   - `git add _posts/YYYY-MM-DD-{제목}.md`
   - `git commit -m "Add blog post: {제목}"`
   - `git push -u origin claude/analyze-git-blog-code-011CUvGjmEd2uuAV6jnuqDZJ`

## 사용자에게 확인할 정보

사용자에게 다음 정보를 요청하세요:
- **제목**: 게시물 제목 (필수)
- **카테고리**: 카테고리명 (기본값: [블로그])
- **태그**: 태그 리스트 (기본값: [])
- **내용**: 마크다운 형식의 본문 내용

만약 사용자가 이미 완성된 마크다운 파일이나 내용을 제공했다면, Front Matter를 확인하고 필요시 보완하여 작성하세요.

## 예시

사용자가 "안녕하세요"라는 내용과 "첫 포스트"라는 제목을 제공하면:

```markdown
---
title: 첫 포스트
date: 2025-11-08 10:00:00 +0900
categories: [블로그]
tags: [일상]
---

안녕하세요
```

파일명: `_posts/2025-11-08-first-post.md`

## 중요사항

- 파일명에는 한글을 사용하지 말고 영문 slug를 사용하세요
- 날짜는 항상 +0900 (서울 시간대) 사용
- 커밋 후 푸시까지 완료하세요
- 푸시 결과를 사용자에게 보고하세요
